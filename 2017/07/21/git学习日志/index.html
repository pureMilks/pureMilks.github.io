git学习日志

git的基本使用

git

分布式版本控制工具

版本控制工具能够管理的内容

  1. 版本控制工具不仅能够保存文件的内容状态

  2. 还能够保存文件的 是否被删除 是否有新增文件

git的基本使用

git的配置命令,这个命令只需要在一台电脑上执行一次

git config –global user.name 名字

git config –global user.email 邮箱

  1. git init

git init命令,是将当前所在的文件夹,用git版本控制工具进行管理,执行这句命令之后,会在当前文件夹中生成一个.git的隐藏文件夹,该文件就是git用来存储每次修改的历史记录的版本仓库

  1. git status

用来查看当前文件夹中,这些文件的状态,如果文件没有被git管理,会有untracked files提示

  1. git add

用来将文件添加到git中,让git进行管理

git add 文件名 文件名

git add * 将所有的文件全部添加到git中进行管理

git add -A 将所有的文件全部添加到git中进行管理

  1. git commit -m “提示信息”

将当前的文件状态保存一份到版本库中,生成一个新的版本,历史记录中会有这个版本的记录

  1. git log

用来查看所有已经保存的版本的内容

  1. git reset –hard 版本号

将代码恢复到指定的版本

  1. git checkout 文件名 将文件从暂存区恢复到工作目录

git原理

分区

  1. 工作目录

  2. 暂存区 在.git文件夹下面

  3. 版本库 在.git文件夹下面

文件的状态

untracked 文件新建之后,并未受过管理就是未追踪状态

modified 文件被修改了

deleted 文件被删除了

staged 已暂存当使用git add 命令将文件添加到暂存区之后,这个文件就处于staged

commited 已提交,当使用git commit提交之后,就是已提交状态

git对于文件的管理,其实就是调用各个命令,将文件在3个分区之间来回传递的过程

使用github进行静态页面托管

github

全球最大的源代码托管平台

github和git的关系

git可以用来获取github上的代码,还可以将自己的代码推送到github上

git可以看做是操作github的一个工具

github静态页面托管服务

一个账户下面只能有一个项目做静态页面托管

  1. 注册github账号

  2. 创建一个仓库,网页右上角一个加号,点击加号,再点击new repository

  3. Repository name要设置成自己的用户名.github.io

  4. 点击creat repository

  5. 复制https的链接

  6. 将本地文件夹使用git进行管理,进行一次提交

  7. 使用git push命令,将本地文件夹中的内容全数推送到github中新创建的仓库中

git push https://github.com/avakpan/123.git master

git push 将本地的仓库推送到github上

  1. 在浏览器地址栏中，直接输入自己的用户名.github.io 就可以访问到自己的网站了
